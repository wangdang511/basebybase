Preferences
Base-By-Base allows users to customize how the application will display alignmed sequences, as well as how it will behave under certain conditions. Below is a list of preferences that are available to the user from within Base-By-Base via the 'Preferences' option in the Edit menu. Configuration information is stored in a file called bbbase.config which is created in the .bbb directory in the users home directory.

Frame X: AA Translation (with genes) (true,false) When set to true, Base-By-Base will display a translation of each sequence in the alignment for the given frame. If gene features have been loaded into BBB, these will also be displayed in the appropriate frame.
Hilight Start/Stop Codons with Arrows (true,false) If this value is set to true (Checked on), then start and stop codons will be indicated in the three frame translation of sequences. Start codons will be indicated by a green arrow over the amino acid and stop codons will be indicated by a red arrow.
Show AA Arows('===>') (true,false) When set to on, BBB will insert arrows between the amino acid labels in the three frame translation. This gives the indication for where each amino acid is coded in case there are large gaps. For more information on the three frame translation see the Display section of the help
Show events and differences (true, false) When set to true, BBB will display a row of difference indications between each sequence.
Show gapped position scale (true, false) This shows or hides the gapped position scale. This the scale of positions in relation to the alignment, not any particular sequence.
Show sequence position scale (true, false) This shows or hids the sequence position scale. This is the scale of sequence positions for each sequence. This scale will be different for each sequence containing different numbers of bases and gaps.
Show user comments/annotations (true, false) This allows users to display or hide comments that may have been added to sequences in the alignment.
Propogate edits to selected sequences (true, false) This setting indicates whether or not an edit action maee on one sequence should be run on each (and only) selected sequence in the list.
Propogate selections to selected sequences (true, false) This setting indicates whether or not a select action maee on one sequence should be run on each (and only) selected sequence in the list.
Warn when substituting aligned sequence (true, false) If set to false, users will not be warned of the possible consequences of inserting manually aligned sequence segments back into the main alignment.
Warn when exiting application (true, false) When set to false, users will no be asked if they are sure they would like to quit Base-By-Base

VOCs Db Connection

Base-By-Base allows users to connect to any Viral Orthologous Clusters database they wish to import gene data and make alignment requests. The Viral Orthologous Clusters (Also known as Poxvirus Orthologous Clusters) homepage is at https://4virology.net/tools/VOCS.

Each database connection has 5 different properties that must be known: the DB name, the user, the password, the host and port of the VOCs server. These properties are stored in a file in the .bbb/db-POCS directory in each users home directory. There may be as many files in this directory as you wish, each with different connection properties. An example file is avaliable HERE. Simply modify this file to connect to the appropriate database and drop this into the .bbb/db-POCS directory in your home directory.

Provided with BBB are sample connections to our main databases. These are the database for the Poxvirus Bioinformatics Resource, and the database for the SARS Bioinformatics Suite.
